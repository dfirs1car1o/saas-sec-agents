# Apex Script Proposals — sfdc-expert

This directory contains read-only Apex script proposals generated by the
**sfdc-expert** agent. These scripts surface data that the sfdc-connect
REST/SOQL collector cannot reach.

## Usage Rules

1. **Human review required.** Do not execute any script without reading it fully.
2. **System Administrator only.** Run in Setup > Developer Console > Execute Anonymous.
3. **Read-only by design.** All scripts use `System.debug()` only — no DML, no callouts.
4. **Paste output back.** After execution, paste the debug log output into your Claude
   Code session or provide it as an input file. sfdc-expert will enrich the finding.

## Naming Convention

```
<control_id>_<date>.apex

Examples:
  SBS-ACS-005_2026-03-03.apex
  SBS-DATA-001_2026-03-03.apex
```

## Safety Checklist (verify before running)

- [ ] First line is `// -- READ-ONLY --`
- [ ] No `insert`, `update`, `delete`, `upsert`, `merge`, or `undelete` statements
- [ ] No `Database.executeBatch()`, `System.scheduleBatch()`, or `Http.send()`
- [ ] Every SOQL query has a `LIMIT` clause
- [ ] Script queries only configuration metadata (Profile, PermissionSet, ConnectedApp, etc.)

## Approval Flow

```
sfdc-expert stages script here
        ↓
Human reviews + approves
        ↓
Human runs in Anonymous Apex console
        ↓
Human pastes debug output back to session
        ↓
sfdc-expert updates finding with observed_value and revised expert_notes
```

Scripts in this directory are **not** committed to CI — they are working artefacts
for the current assessment session. Archive or delete after each review cycle.
